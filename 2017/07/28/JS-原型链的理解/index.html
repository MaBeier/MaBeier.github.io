JS-原型链的理解

Jul 28, 2017 | 阅读

原型链

一切引用类型来自obj，继承obj.

确定原型和实例之间的关系:

实例 instanceof 原型；
原型.prototype.isPrototypeOf(实例)

原型链的问题：

可能造成，实例是其他函数的原型。其下的实例共享所有属性方法。造成污染。
综上实践中很好单独使用原型链

解决方法

借用构造函数：

function SuperType(){
    this.colors=['red','blue','green']
    }

function SubType(){
    //继承了SuperType
    SuperType.call(this);
}
var instance1=new SubType();
instance1.colors.push()    //不影响其他实例

组合集成：

使用原型链实现对原型属性和方法的集成，而通过借用构造函数来实现对实例属性的继承
